# Voice ID 模块测试方案 — 执行者手册

本文档为 Voice ID 模块测试的执行指南，涵盖注册、录音、评估、调优、自动化、数据管理和风险控制的详细操作步骤。

---

## 1. 注册（Enrollment）方案

### 三种方案对比

**方式 A：使用场景 1（干净环境）录音注册**
- 优点：环境静音，声纹清晰，易获得高质量模板；流程简单。
- 缺点：仅代表理想条件声纹，对 AEC 干扰或多说话人场景缺乏适应性。

**方式 B：每人另录一段干净语音注册**
- 优点：可独立于测试录音进行，录音时可更专注。
- 缺点：模板仅涵盖干净语音，不包含噪音变化信息。

**方式 C：混合注册（场景 1、2、3 各取一段）** ✅ **推荐采用**
- 优点：声纹模板包含干净语音、AEC 失真和交叉干扰下的变化，提升实际场景鲁棒性。
- 缺点：录音量大，每人需多场景录制；混合异构数据可能略微降低对纯净语音的匹配精度。

**推荐理由**：本次测试目标要求所有场景下都有高识别率，混合注册能提前使声纹模型见到干扰条件中的变化，从而更有可能在测试中保持性能。场景 1 的干净语音亦包含在内，保证纯净环境下的准确率。

### 具体流程

1. 每位受试者在三个场景下各录制约 10–20 秒的语音，可分多段进行。
2. 录音内容选择与测试文本一致的固定句子和自由话题。
3. 使用 Pipecat 提供的注册接口将音频导入 VoiceIDProcessor，系统会提取 256 维归一化声纹向量并生成模板。
4. **每人注册音频总长建议 30 秒以上**，以获得稳定模板。

## 2. 录音协议设计

### 固定文本示例

- **「海纳百川，有容乃大；壁立千仞，无欲则刚。」** — 含多种声母和韵母，适合提取声韵特色。
- **「春风得意马蹄疾，一日看尽长安花。」** — 古诗示例，语调起伏明显。
- 固定文本应保持发音标准，长度控制在 5–10 秒内。

### 自由说话

可设定话题方向，例如：
- 个人简介（名字、职业、兴趣）
- 最近的旅行经历
- 喜欢的电影 / 音乐
- 日常生活描写

让受试者自然叙述，各自录制 15–30 秒，涵盖普通用词及发音变化。

### 每段时长建议

单段录音应保持在 **5–15 秒** 范围，以平衡信息量与数据处理需求。句子过短则声纹特征不足，过长则增加后期处理复杂度。

### 录音批次安排

按场景分批进行，流程示例：

1. **第一场景（clean）**：所有 6 人依次录制其固定文本及自由话音，完毕后稍作休息。
2. **第二场景（aec_distorted）**：在开启 AEC 处理的情况下，所有 6 人再次按顺序录制固定文本及自由话音。
3. **第三场景（cross_talk）**：模拟交叉干扰环境，所有 6 人录制固定文本及自由话音。

每个场景大约安排 2–3 小时完成。若一次性连续测试压力大，也可分 2 天完成。每个受试者每场景录制 5–10 段，根据具体时间和人手情况调整。

### 文件命名规范

```
{speakerID}_{scene}_{segmentID}_{type}.wav
```

- `speakerID`：说话者编号，如 S01、S02
- `scene`：场景标签，如 clean、aec_distorted、cross_talk
- `segmentID`：段落号码，两位数编号
- `type`：内容类型，`fixed`（固定文本）或 `free`（自由说话）

例如：`S03_cross_talk_07_free.wav` 表示 3 号说话人在交叉干扰场景下的第 7 段自由话语音频文件。

### 场景 3 干扰控制

建议使用**播放固定背景音频**的方式控制交叉干扰：

1. 事先录制一段包含 1–2 人对话或环境语音的音频文件。
2. 录制时通过音箱播放，确保每次测试使用同一段背景音。
3. 背景音量设定在与说话者口语音量相近或略低的水平（可用分贝计测试调校）。
4. 背景音源位置固定：离说话者 0.5–1 米处，高度与麦克风相当。
5. 记录播放音量、设备位置等参数，以保证每次测试条件一致。

## 3. 评估指标体系

### 核心指标

- **FAR（误接受率）**：非目标说话人被系统误认为目标的比例。`FAR = 假接受次数 / 总验证次数 × 100%`
- **FRR（误拒绝率）**：目标说话人被系统错误拒绝的比例。`FRR = 假拒绝次数 / 总验证次数 × 100%`
- **EER（等错误率）**：FAR 与 FRR 相等时的错误率值，用于比较系统在最优阈值下的综合性能。
- **Top-1 准确率**：`正确识别样本数 / 总样本数 × 100%`

### 样本规模可靠性

本次测试大约有 **450–900 个比对样本**（6 人 × 3 场景 × 每人 5–10 段 × 对手数 5），在此规模下计算出的 FAR / FRR 等指标具有较好的统计意义。可以使用 95% 置信区间评估指标置信度，误差通常在几个百分点内。

### 场景维度对比

对同一说话人，对比其在三种场景下的 Top-1 准确率、FAR、FRR 等指标。可用折线图或柱状图展示场景间性能差异，有助于定位场景造成的性能下降。若某一场景普遍低于其他，可重点分析该场景下的干扰因素。

### 混淆矩阵

建立 **6×6 说话人混淆矩阵**：
- 行表示真实说话人（S01–S06），列表示系统识别结果。
- 矩阵元素 M[i,j] 记录真实为 Si 但被识别为 Sj 的次数。
- 理想情况下对角线元素接近最大值，非对角元素接近 0。
- 若发现非对角元素较高，说明 Si 和 Sj 两人的声纹相似或模型混淆度高，需要进一步调整参数或收集更多训练样本。

### 通过标准

| 场景 | Top-1 准确率门槛 |
|------|-----------------|
| 干净场景 | > 95% |
| AEC 场景 | > 90% |
| 交叉干扰 | > 85% |

若某个场景未达标，需回到参数调整或录音质量检查环节进行迭代改善。

## 4. WeSpeaker 参数调优路线图

### 参数清单（按影响程度排序）

| 参数 | 说明 |
|------|------|
| **SIM_THRESHOLD**（相似度阈值） | 最重要参数，直接影响接受 / 拒绝平衡，改变 FAR 与 FRR |
| **度量方法** | 余弦相似度 vs. PLDA。WeSpeaker 生成 256 维向量，可选择使用余弦相似度或 PLDA 进行匹配，不同方法下最佳阈值可能不同 |
| **BUFFER_SECS**（缓冲时长） | 控制每次语音段的最小长度，过短可能损失说话信息，过长可能包含多余静音或背景 |
| **GATE_TIMEOUT**（门控超时） | 控制语音停止后等待 ASR 门控输入的时间，较大值可处理长尾语音，较小值可提高响应速度 |
| **UNKNOWN_POLICY** | 当系统判定非注册说话人时的行为（如 pass 继续处理或 mute 静音），不直接影响识别精度，但影响交互体验 |
| **其他** | VAD 灵敏度、增益控制等音频前处理参数 |

### 调优顺序与方法

依次进行粗到细的调整：

1. **调整 SIM_THRESHOLD**：保持其他参数不变，在干净语音下跑测试，观察 FAR / FRR 随阈值变化。**提高阈值会降低 FAR 但提高 FRR；降低阈值则相反**。选取使 FAR 和 FRR 达到预定平衡点的阈值。

2. **切换度量方法**：在最佳阈值基础上，将匹配方式从余弦改为 PLDA，重新测试所有场景。对比 Top-1 准确率和错误率变化。如果 PLDA 整体提升，则考虑使用 PLDA；否则保留余弦。

3. **调整 BUFFER_SECS**：在实际录音中检查系统是否截断了较短的语音段。若是，适当增大缓冲时长；如果系统处理时间过长或包含了过多沉默，则可减少该参数。

4. **微调 UNKNOWN_POLICY 和 GATE_TIMEOUT**：观察当输入非注册语音或噪音时系统行为。如果更注重完整对话，可以设为 pass；若需保证安全，设为 mute 也可。GATE_TIMEOUT 可根据实际语音停顿情况调整，避免误提前停止或延迟输入。

### 参数交互效应

部分参数互有耦合。例如，不同度量方法需要不同阈值；BUFFER_SECS 与 SIM_THRESHOLD 的配合也会影响结果。调整时宜一个参数一个参数地改动，以判断单独影响，然后再考虑参数组合。

## 5. Auto-Research 集成方案

### 搜索空间设计

为每个可调参数设定范围：

| 参数 | 搜索范围 | 步长 |
|------|----------|------|
| SIM_THRESHOLD | 0.5 – 0.95 | 0.01 |
| BUFFER_SECS | 0.5 – 2.0 秒 | 0.1 |
| GATE_TIMEOUT | 1 – 10 秒 | 1 |
| UNKNOWN_POLICY | {pass, mute} | — |
| 相似度度量 | {cosine, PLDA} | — |

根据经验或初步试验缩小范围，加快搜索效率。

### 目标函数

定义模型性能评价指标，例如使用**加权准确率**：`0.7 × (Top-1 准确率) + 0.3 × (1 - FRR)`。或者直接最大化 Top-1 且要求 FRR 低于设定值。目标函数需凸显高准确和低误拒。

### 自动化流程

编写脚本配合 AutoResearch 代理执行如下流程：

1. **加载参数**：读取当前实验的一组参数设定。
2. **应用设置**：将参数写入配置文件或通过 API 下发到 Voice ID 模块。
3. **执行测试**：运行批量测试集（约 450–900 条语音对），收集每对的相似度分数和预测结果。
4. **计算指标**：统计 FAR、FRR、EER 和识别准确率，并计算定义的目标函数值。
5. **反馈结果**：将参数配置和评价结果回传给 AutoResearch，作为下一轮参数生成的依据。
6. **迭代**：重复上述步骤，直至完成预定轮数或检测到收敛。

> AutoResearch 每次实验可设定固定时间（例如 5 分钟），大约每小时可进行 12 次实验。考虑到我们每次实验可能需要运行完整测试集，实际每小时实验次数可能更少。初步估计，进行**数十到百次**迭代可基本覆盖多数组合。

### 收敛判断

若经过多轮搜索后，目标指标（如 Top-1 准确率）提升幅度很小或指标达到预定上限，则可以认为参数已接近最优。此时可停止搜索，采用当前最佳参数。

## 6. 测试数据管理

### 测试数据表模板

建立一个表格（例如 CSV 格式）记录每次对话的结果，字段包含：

```
speaker_id | scene | segment_id | content_type | true_speaker | predicted_speaker | score | is_correct
```

示例：

| speaker_id | scene      | segment_id | content_type | true_speaker | predicted_speaker | score | is_correct |
| ---------- | ---------- | ---------- | ------------ | ------------ | ----------------- | ----- | ---------- |
| S01        | clean      | 001        | fixed        | S01          | S01               | 0.87  | True       |
| S01        | cross_talk | 005        | free         | S01          | S02               | 0.60  | False      |

`score` 列记录相似度分数（如余弦值），`is_correct` 为识别是否正确（True / False 或 1 / 0）。

### 自动评估脚本骨架

```python
import pandas as pd
import numpy as np
from sklearn.metrics import roc_curve, confusion_matrix
import seaborn as sns
import matplotlib.pyplot as plt

# 1. 读取测试结果
df = pd.read_csv('test_results.csv')

# 2. 计算 Top-1 准确率
accuracy = np.mean(df['is_correct'])
print(f'Accuracy: {accuracy*100:.2f}%')

# 3. 混淆矩阵计算与绘制
labels = sorted(df['true_speaker'].unique())
cm = confusion_matrix(df['true_speaker'], df['predicted_speaker'], labels=labels)
sns.heatmap(cm, annot=True, fmt='d',
            xticklabels=labels, yticklabels=labels)
plt.xlabel('Predicted')
plt.ylabel('True')
plt.title('Confusion Matrix')
plt.show()

# 4. 计算 FAR, FRR, EER（二分类情境示例）
# 将多说话人问题转为二分类：将目标说话人视为正类，其他为负类
y_true = df['true_speaker'] == df['predicted_speaker']
scores = df['score']
fpr, tpr, thresholds = roc_curve(y_true, scores)
fnr = 1 - tpr
eer_threshold = thresholds[np.nanargmin(np.abs(fnr - fpr))]
eer = fpr[np.nanargmin(np.abs(fnr - fpr))]
print(f'EER: {eer*100:.2f}%')

# 5. 输出报告
# 将以上结果整合成表格或文本报告形式存档
```

上述代码演示了如何读取 CSV 结果、计算准确率、绘制 6×6 混淆矩阵（使用 Seaborn 热力图）以及基于 ROC 曲线估计 EER。实际使用时可能需要根据数据格式做细节调整，例如处理多说话人的标签方式等。

## 7. 风险清单与缓解方案

| 风险 | 缓解措施 |
|------|----------|
| **录音质量差异**：不同被测者音量大小、吐字清晰度不一可能导致声纹特征不均。 | 统一使用同型号麦克风，进行录音前的增益校准；录音过程中观察波形或监听检查，必要时重新录制。需注意环境噪音、麦克风与说话者距离等外部因素对结果的影响，因此录音时尽量保持环境安静、距离一致。 |
| **AEC 扭曲不可控**：不同回声消除算法版本可能引入不一样的失真。 | 测试前确认并固定使用的 AEC 版本与参数；对于可能的升级或更换，在新算法上重新评估系统性能；可考虑在注册阶段模拟 AEC 处理以补偿差异。 |
| **场景 3 干扰不可重复**：现场人声干扰易受语音内容和音量变动影响。 | 使用预录制的背景对话音频并保持播放设定一致，以固定噪音来源；在每次录制前校准背景音量和设备位置，并记录参数以保证复现。 |
| **测试样本量偏小**：每人每场景只有 5–10 段语音可能导致统计波动。 | 可在分析结果时计算置信区间，并在评估不稳定时增加录音次数；如需更高置信度，可考虑录制更多段或多次测试取平均。 |
| **嵌入式性能限制**：类树莓派设备算力有限，执行 WeSpeaker 嵌入和比对可能较慢。 | 监测运算延迟，必要时对模型进行轻量化（如量化、维度减小）或使用多线程；优化代码与配置，并避免在录音期间同时执行其他高负载操作。如果推理时延成问题，可考虑预先并行处理批量录音数据来提高效率。 |
